# Causal Mechanism Library

## Systems Thinking Framework

This document provides a structured list of **causal mechanisms** used in systems thinking and structural analysis.

Causal mechanisms represent **fundamental processes that drive system behavior**.
They explain how individual interactions within a system produce larger patterns.

In the systems thinking framework used in the Elite Think Tank Mastery program, the analytical hierarchy is:

```
Events
↓
Causal Mechanisms
↓
System Archetypes
↓
System Behavior
↓
Strategic Intervention
```

---

# What is a Causal Mechanism?

A **causal mechanism** is a structural process that explains **how and why a system produces certain outcomes**.

Mechanisms describe **cause-and-effect relationships** operating within a system.

They operate at the **micro-level of system dynamics**, while system archetypes represent **recurring macro-level patterns**.

---

# Core Causal Mechanisms

These mechanisms form the core structural drivers frequently observed across systems.

| Mechanism                | Definition                                                 |
| ------------------------ | ---------------------------------------------------------- |
| Growth                   | Increase in scale, demand, or activity within a system     |
| Coordination Complexity  | Interactions increase as the system grows                  |
| Decision Bottleneck      | Decisions depend on a limited number of actors             |
| Network Effects          | System value increases as more participants join           |
| Infrastructure Lag       | Capacity expansion fails to keep up with demand            |
| Shared Resource Pressure | Multiple actors compete for limited resources              |
| Competitive Response     | Actors react to competitors’ actions                       |
| Risk Aversion            | Failures lead to increased control and restrictions        |
| Short-Term Fix           | Temporary solutions replace structural fixes               |
| Structural Inertia       | Existing structures resist change                          |
| Feature Expansion        | Continuous feature additions increase complexity           |
| Dependency               | Critical resources or knowledge concentrated in few actors |

# Organizational Mechanisms

These mechanisms commonly appear within organizations and institutions.

| Mechanism               | Definition                                             |
| ----------------------- | ------------------------------------------------------ |
| Incentive Misalignment  | Individual incentives conflict with system goals       |
| Ownership Dilution      | Responsibility becomes unclear as systems scale        |
| Bureaucratic Expansion  | Rules and procedures increase over time                |
| Communication Overhead  | Information exchange increases with system size        |
| Knowledge Concentration | Expertise resides in a few individuals                 |
| Coordination Delay      | Teams require additional synchronization before acting |

---

# Technology System Mechanisms

These mechanisms frequently appear in technology systems and platforms.

| Mechanism               | Definition                                            |
| ----------------------- | ----------------------------------------------------- |
| Technical Debt          | Accumulated shortcuts create system fragility         |
| Architecture Complexity | System design becomes difficult to maintain           |
| Platform Dependency     | Ecosystems become dependent on a central platform     |
| Integration Complexity  | Many system components require coordination           |
| Scaling Constraint      | Infrastructure struggles to support increasing demand |

---

# Market and Economic Mechanisms

These mechanisms commonly appear in competitive markets.

| Mechanism          | Definition                                           |
| ------------------ | ---------------------------------------------------- |
| Price Competition  | Firms lower prices to compete                        |
| Innovation Race    | Firms compete through technological advancement      |
| Market Saturation  | Demand growth slows as markets mature                |
| Entry Barrier      | Structural advantages prevent new competitors        |
| Resource Advantage | One actor gains disproportionate access to resources |

---

# Infrastructure and Resource Mechanisms

These mechanisms appear in public systems and infrastructure networks.

| Mechanism            | Definition                                   |
| -------------------- | -------------------------------------------- |
| Capacity Constraint  | Infrastructure limits system growth          |
| Demand Growth        | Increasing demand places pressure on systems |
| Resource Depletion   | Shared resources become scarce               |
| Congestion Formation | System usage exceeds system capacity         |

---

# Example Mechanism Chain

A typical causal chain within an organization:

```
Growth
→ Team Expansion
→ Coordination Complexity
→ Decision Bottleneck
→ Slower Execution
```

This mechanism chain produces the system archetype:

```
Limits to Growth
```

---

# Mechanisms and System Archetypes

Mechanisms combine to produce **recurring structural patterns called archetypes**.

Example:

```
Growth
→ Coordination Complexity
→ Decision Bottleneck
```

Archetype:

```
Limits to Growth
```

Another example:

```
Network Effects
→ More Users
→ More Creators
→ More Content
→ More Users
```

Archetype:

```
Success to the Successful
```

# Why a Mechanism Library Matters

A causal mechanism library helps system thinkers:

* identify structural drivers of system behavior
* construct causal chains
* recognize archetypal patterns
* diagnose organizational problems
* identify leverage points for intervention

Mechanisms therefore act as the **building blocks of system diagnosis**.
