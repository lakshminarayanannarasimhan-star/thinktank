# System Archetype Library

## Systems Thinking Framework

This document provides a structured list of **system archetypes** used in systems thinking and structural analysis.

System archetypes represent **recurring structural patterns that emerge in complex systems**.

Within the systems thinking framework used in the Elite Think Tank Mastery program, the analytical hierarchy is:

```
Events
↓
Causal Mechanisms
↓
System Archetypes
↓
System Behavior
↓
Strategic Intervention
```

---

# What is a System Archetype?

A **system archetype** is a recurring structural pattern that emerges when multiple causal mechanisms interact within a system.

Archetypes represent **common system behavior patterns observed across organizations, markets, technology systems, and infrastructure systems**.

While **mechanisms explain how systems evolve**, archetypes describe **the structural pattern produced by those mechanisms**.

---

# Core System Archetypes

These archetypes appear repeatedly across many types of systems.

| Archetype                            | Description                                                       |
| ------------------------------------ | ----------------------------------------------------------------- |
| Limits to Growth                     | Growth eventually slows due to emerging constraints               |
| Success to the Successful            | Early success reinforces further advantage                        |
| Growth & Underinvestment             | System growth exceeds investment in capacity                      |
| Escalation                           | Competing actors continually respond to each other’s actions      |
| Shifting the Burden                  | Short-term fixes replace structural solutions                     |
| Tragedy of the Commons               | Shared resources become overused                                  |
| Control Expansion                    | Failures lead to increased rules and restrictions                 |
| Success Lock-in                      | Past success creates structural resistance to change              |
| Feature Creep                        | Continuous feature additions increase system complexity           |
| Dependency / Single Point of Failure | System stability depends on a small number of critical components |

---

# Archetype Descriptions and Examples

## Limits to Growth

Growth initially accelerates but eventually slows as constraints emerge.

Example:

```
Growth
→ Team Expansion
→ Coordination Complexity
→ Decision Bottleneck
→ Slower Execution
```

Outcome:

System performance declines as the system becomes more complex.

---

## Success to the Successful

Early advantages reinforce themselves and lead to further success.

Example:

```
Users
→ More Creators
→ More Content
→ More Users
```

Outcome:

Dominant platforms or organizations emerge.

---

## Growth and Underinvestment

Demand increases faster than system capacity expands.

Example:

```
Population Growth
→ Infrastructure Investment Lag
→ Capacity Shortage
→ Traffic Congestion
```

Outcome:

System performance deteriorates.

---

## Escalation

Two actors continually respond to each other’s actions.

Example:

```
Company A lowers price
→ Company B responds
→ Price War
→ Industry Margin Decline
```

Outcome:

Both actors suffer long-term losses.

---

## Shifting the Burden

Short-term solutions replace deeper structural fixes.

Example:

```
System Failure
→ Temporary Fix
→ Root Cause Remains
→ Repeated Failures
```

Outcome:

System fragility increases over time.

---

## Tragedy of the Commons

Shared resources become overused by individual actors.

Example:

```
Shared Infrastructure
→ Increasing Usage
→ Resource Overload
→ System Slowdown
```

Outcome:

System capacity becomes exhausted.

---

## Control Expansion

Failures cause organizations to introduce additional rules and controls.

Example:

```
Operational Failure
→ Risk Aversion
→ New Approval Layers
→ Decision Bottleneck
```

Outcome:

Innovation and execution slow.

---

## Success Lock-in

Past success creates resistance to change.

Example:

```
Successful Product
→ Structural Inertia
→ Lack of Innovation
→ Market Shift
```

Outcome:

Organizations fail to adapt to new environments.

---

## Feature Creep

Continuous addition of features increases system complexity.

Example:

```
Customer Requests
→ Feature Expansion
→ Architecture Complexity
→ Maintenance Difficulty
```

Outcome:

Product becomes difficult to maintain and evolve.

---

## Dependency / Single Point of Failure

Critical system knowledge or functionality is concentrated in a few actors.

Example:

```
Critical System Knowledge
→ Few Experts
→ High Dependency
→ System Risk
```

Outcome:

System stability becomes fragile.

---

# Archetypes and Causal Mechanisms

System archetypes emerge when causal mechanisms interact.

Example:

```
Growth
→ Coordination Complexity
→ Decision Bottleneck
```

Archetype:

```
Limits to Growth
```

Another example:

```
Network Effects
→ Increasing User Participation
→ Ecosystem Expansion
```

Archetype:

```
Success to the Successful
```

# Why Archetypes Matter

System archetypes help analysts:

* recognize recurring structural patterns
* anticipate system outcomes
* understand system dynamics
* diagnose complex problems

Because archetypes appear repeatedly across systems, they allow analysts to **apply insights learned in one domain to another**.
